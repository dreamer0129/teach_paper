# 计算机教育期刊写作助手

## 概述

本技能帮助作者撰写符合"计算机教育"期刊要求的学术论文，从选题到成稿提供全程指导。基于期刊官方投稿指南和已录用论文分析，确保论文格式规范、内容专业、易于录用。

## 快速参考

| 任务 | 操作 |
|------|------|
| 新论文选题 | 分析当前教育热点，提供选题建议 |
| 结构规划 | 按期刊要求设计章节大纲 |
| 内容写作 | 提供各章节写作指导和示例 |
| 格式验证 | 检查字数、摘要、引用格式 |
| 生成文档 | 自动生成符合模板的Word文件 |

## 工作流程

### 1. 选题与规划阶段

**触发场景**：
- "我想写一篇计算机教育期刊论文，有什么选题建议？"
- "AI在教育中的应用有哪些研究方向？"
- "给我推荐几个热门的论文主题"

**操作流程**：
1. 分析用户研究方向
2. 参考已录用论文主题（AI赋能、课程改革等）
3. 提供3-5个具体选题建议
4. 设计章节大纲
5. 确认选题后进入内容写作

**常见选题方向**：
- AI+X系列：AI赋能各门课程改革
- 大模型应用：大语言模型在教学中的应用
- 教学创新：混合式教学、项目式学习等
- 课程思政：思政与专业教育融合

### 2. 内容写作阶段

**触发场景**：
- "帮我写论文摘要"
- "指导我写论文正文"
- "如何写参考文献"

**各部分要求**：

#### 题目（≤20汉字）
- 要求：简洁明了，反映研究内容
- 避免使用冗余词："的研究"、"的探索"等
- 示例："AI赋能微处理器设计课程改革"

#### 摘要（200-300字，第三人称）
- 结构：研究目的 + 方法 + 结果 + 结论
- 句式："为了...，对...进行了研究"
- 避免第一人称："本文"、"作者"
- 示例："为了解决计算机公共基础课程教学中理论与实践脱节的问题，对基于大模型的Python教学内容进行了创新设计..."

#### 关键词（3-8个）
- 格式：分号或空格分隔
- 选择：主题词 + 方法词 + 对象词
- 示例："人工智能；教育改革；课程思政；计算机教育"

#### 正文
- 引言：背景、现状、问题、创新点
- 主体：改革方案、实施过程、效果分析
- 结论：成果总结、创新点、不足、展望

### 3. 格式验证阶段

**触发场景**：
- "检查一下我的论文格式"
- "验证摘要是否符合要求"
- "统计字数是否达标"

**验证项目**：

#### 自动验证脚本
- `scripts/validate/validate_abstract.py`：检查摘要字数、第三人称、内容完整性
- `scripts/validate/validate_wordcount.py`：统计总字数（正文+图表+参考文献+摘要）
- `scripts/validate/validate_title.py`：检查题目长度（≤20汉字）
- `scripts/validate/validate_references.py`：检查引用格式是否正确

#### 生成验证报告
``=== 格式验证结果 ===
[✓] 摘要：256字，符合200-300字要求
[✓] 题目：16字，符合≤20字要求
[✓] 字数：5832字，符合≤6000字要求
[!] 参考文献：第3条格式不正确，请检查期刊格式
```

### 4. 文档生成阶段

**触发场景**：
- "生成Word文档"
- "创建符合模板的论文"
- "导出最终版"

**生成流程**：
1. 使用官方模板：`assets/计算机教育-论文模版.docx`
2. 通过unpack解包为XML格式
3. 注入论文内容到XML
4. 重新打包生成最终Word文档

**输出文件**：`计算机教育论文_最终版.docx`

## 详细指南

### 选题指导

#### 高价值选题特征
- **技术前沿**：AI、大模型、生成式AI等
- **教育需求**：解决教学实际问题
- **创新性**：新方法、新技术、新模式
- **实践性**：有具体实施方案和效果数据

#### 选题决策树
```
有具体课程？ → 是 → AI赋能XX课程改革
               → 否 → 有教学方法？ → 是 → 教学模式创新
                            → 否 → 有技术工具？ → 是 → 技术应用
                                                    → 否 → 理论研究
```

### 内容写作建议

#### 摘要写作模板
```
为了[解决XX问题]，对[XX领域]进行了[研究/分析]。通过[XX方法/技术]，实现了[XX目标]。实验结果表明[XX效果]，表明该方法[XX价值]。
```

#### 正文写作要点
1. **问题导向**：清晰指出当前存在的问题
2. **方案具体**：提供详细的实施步骤和方法
3. **数据支撑**：用实验数据证明效果
4. **对比分析**：与传统方法的优势对比
5. **推广应用**：说明推广价值和条件

### 参考文献注意事项

1. **格式严格**：严格按照reference-formats.md要求
2. **数量适当**：10-20篇为宜，包含最新文献
3. **质量保证**：优先选择核心期刊、权威出版物
4. **引用规范**：文中引用按顺序编号

## 常见问题与解决方案

### Q1: 如何确定创新点？
**A**: 从以下角度寻找创新点：
- 技术创新：新技术应用、新方法设计
- 方法创新：教学模式、评价方式创新
- 理论创新：新理念、新框架构建
- 应用创新：新场景、新领域拓展

### Q2: 字数超限怎么办？
**A**:
- 合并重复内容
- 删除冗余描述
- 精简图表说明
- 压缩参考文献数量

### Q3: 如何体现实践效果？
**A**:
- 对比实验数据
- 学生反馈统计
- 教学效果分析
- 推广应用案例

### Q4: 参考文献格式错误？
**A**:
- 检查文献标识[J][M][C]等
- 确认标点符号使用
- 验证作者姓名格式
- 核对期刊名称缩写
